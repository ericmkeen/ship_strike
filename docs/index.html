Eric Keen

This vignette demonstrates the utilities within the R package shipstrike by stepping through the analysis for Keen et al., “A simulation-based tool for predicting whale-vessel encounter rates” (submitted to Ocean & Coastal Management). This study is carried out in close collaboration with the Gitga’at First Nation, North Coast Cetacean Society, and WWF-Canada as part of a research project named SWAG: Ships, Whales, and Acoustics in Gitga’at waters.

Marine traffic data

Formatting archival AIS data

Subsequent steps will require AIS data to be formatted into a data.frame meeting the following criteria:

  • Each row contains details for a single vessel.
  • There is a column named vid, containing a unique identifier for each vessel (e.g., MMSI).
  • There is a column named type, containing the category or class of vessel.
  • There is a column named speed, containing speed-over-ground in knots.
  • There is a column named length, containing vessel length in meters.
  • There is a column named width, containing vessel beam width in meters.
  • There is a column named draft, containing vessel draft at this timestamp.
  • There is a column named datetime, with date/time, in UTC, formatted as yyyy-mm-dd hh:mm:ss.
  • There are columns named x and y for longitude and latitude, respectively, in decimal degrees.

Determine the sun angle for each fake AIS fix:

vgrid_sim$sun <- vessel_sun_angle(vgrid_sim, verbose=FALSE)

Based on sun angle, assign a diel period designation (night or day):

Whales

The parameters that characterize a whale in the encounter simulator can be defined as either a single value (e.g., 20 meters), or as a distribution of values from which to draw random values. In this example, we will do the latter.

Define the size of the distributions you will use:

n <- 1000

These distributions will be truncated-normal distributions, to ensure that no value is unrealistically small or large.

Whale velocity

Whale velocity should be provided as meters per second. We draw values from the acoustic tracks of fin whales from the same study area in Hendricks et al. (2021).

v.whale = truncnorm::rtruncnorm(n,
                                a=0,
                                b=2.63,
                                mean=1.3611,
                                sd=.5)

Whale directivity

Whale directivity, which we define in our paper as the standard deviation of change in course heading from one minute to the next, is also drawn from Hendricks et al. (2021). In that paper, 62 acoustic tracks of at least 30 minutes duration were analyzed. Refer to prep_delta_sd.R for details on how this was determined. Note that, in this analysis, the standard deviation of course changes were log-transformed so that they followed a Gaussian distribution.

# Produce distribution of log-transformed values
delta.sd = rnorm(n, 
                 mean = 1.0424,
                 sd = 0.82477)

# Revert from log-transformed to actual SD of course change
delta.sd <- exp(delta.sd)

Whale dimensions

Whale length is drawn from UAV-based photogrammetry measurements for fin whales in our study area, published in Keen et al. (2021).

l.whale = truncnorm::rtruncnorm(n = n,  # samples
                                a = 0,  # min
                                b = 40, # max
                                mean = 18.60375,
                                sd = 1.649138)

To estimate whale widths, we use the ratio of fluke width to body length from Keen et al. (2021).

w.whale = .2074

Encounter rate simulator

Basic example

You can conduct an encounter rate simulation using the shipstrike function. The code below demonstrates differences in various whale directivity values:

par(mfrow=c(1,3))
delta.sds <- c(0,30,80)

for(i in 1:3){
encounters <- 
  encounter_simulator(params.ship=params.ship,
                      v.whale=v.whale,
                      l.whale=l.whale,
                      w.whale=w.whale,
                      delta.sd=delta.sds[i],
                      B=100)
}

Note that full details are only given for runs in which the whale and ship came within one length of the side of the ship; this is done to save time, but note that you may change encounter_simulator() inputs to save more details (see ?encounter_simulator).

The column encounter will have the value 1 if the whale’s rostrum occurs within the ship’s polygon boundary at any time. However, it is possible for a whale to be struck in other scenarios: for example, the ship may strike the whale’s side. To gain an accurate tally of imminent collisions, use the encounter_tally() function in the shipstrike package:

encounter_tally(encounters)$total
#> [1] 8

This function asks whether, in any of the near-misses, the rest of the whale’s body might have intersected with the ship, based on the whale’s length in each run and the whale’s heading at the moment of closest proximity.

The function returns a list with slots for details, and a total slot with the final estimate of close encounters for this simulation run.

Full example

We are now ready to run our encounter rate simulation for a real analysis.

Define the number of iterations you want. We suggest no fewer than 100.

iterations <- 100
runs <- 100 # runs per iteration

Now let ’er rip!

par(mfrow=c(2,2))
p_encounters <- c()
for(i in 1:iterations){

  # Run the simulator
  encounters <- 
    encounter_simulator(params.ship = params.ship,
                        v.whale = v.whale,
                        l.whale = l.whale,
                        w.whale = w.whale,
                        delta.sd = delta.sd,
                        B = runs)

  # Process and tally encounters
  encs <- encounter_tally(encounters)$total
  (p_encounter <- encs / runs)
  p_encounters <- c(p_encounters, p_encounter)
  
  message(i,' :: ',p_encounter)
}

Impacts assessment

To incorporate this encounter rate into a shipping impacts assessment framework, we conduct the following stochastic routine. The tankers object tells us the total kilometers transited during the study window (July 1 - October 31):

km <- summarize_grid(tanker)$km
km
#> [1] 12305

We can treat this number of kilometers as the number of potential co-occurrence opportunities during that timeframe. We pass that value to the shipstrike function outcome(), along with the following probabilities:

  • p_whale: the probability of a whale occurring within a single square kilometer, which is whale density; we provide this as a truncated normal distribution based on the results from Keen et al. (2021);

  • p_encounter: the close encounter probability distribution, which we prepared above;

  • p_surface: the probability of the whale occurring within the near-surface ‘strike zone’ at the time of the close enocounter, which for our purposes here we treat as a single average value based on our review of the literature;

  • p_avoidance: the probability of avoidance, which for our purposes here we treat as a single average value representing the best possible case based on our review of the literature;

  • p_lethality: the probability of mortality in the event of collision, which we treat simply as a single average value drawn from the literature;

Here is a single run of the outcome() function:

outcome(p_whale = truncnorm::rtruncnorm(10000, a=0, mean=0.015,sd=0.0031),
        p_encounter = p_encounters,
        p_surface = 0.62, 
        p_avoidance = 0.66, 
        p_lethality = 0.42,
        km = km,
        iterations = 1)
#>   cooccurrence encounter surface collision mortality
#> .          182        17      12         3         2

This function uses stochastic methods to predict the number of co-occurrence events, close encounters, surface-overlap events, collisions (in the event of failed avoidance), and mortalities. Since this is a stochastic prediction, if we run it again we will get slightly different predictions:

outcome(p_whale = truncnorm::rtruncnorm(10000, a=0, mean=0.015,sd=0.0031),
        p_encounter = p_encounters,
        p_surface = 0.62, 
        p_avoidance = 0.66, 
        p_lethality = 0.42,
        km = km,
        iterations = 1)
#>   cooccurrence encounter surface collision mortality
#> .          169         8       4         3         2

A stochastic approach like this allows the uncertainty inherent to any of the parameters (in this specific example, for instance, we have built uncertainty into p_whale and p_encounter) to propagate into our outcome predictions. To do so, change the iterations argument for the function. Doing so will generate a posterior distribution of predictions; for this demo we will just use 1,000 iterations (the published paper uses 10,000):

# Iteratively predict outcomes
results <- outcome(p_whale = truncnorm::rtruncnorm(10000, a=0, 
                                                   mean=0.015,
                                                   sd=0.0031),
                   p_encounter = p_encounters,
                   p_surface = 0.62, 
                   p_avoidance = 0.66, 
                   p_lethality = 0.42,
                   km = km,
                   iterations = 1000, 
                   verbose = FALSE)

Prepare function for iterative differencing:

pval <- function(val1, val2, B=10000){
  vals1 <- sample(val1,size=B,replace=TRUE)
  vals2 <- sample(val2,size=B,replace=TRUE)
  diffs <- vals1 - vals2

  p1 <- length(which(diffs>0)) / length(diffs) # If significant = val 1 is larger than val2
  p2 <- length(which(diffs<0)) / length(diffs) # If significant = val 2 is larger than val1

  par(mar=c(4.2,4.2,3,1))
  hist(diffs,col="grey70",border="grey60",main="val1 - val2",xlab="Differences")
  abline(v=0,lty=3,col="steelblue")

  return(list(diffs=diffs,p1=p1,p2=p2))
}

Summarize sensitivity analysis results

Stage a helper function:

sensitivity_summary <- function(mrs){
mrsumm <- 
  mrs %>% 
  group_by(vari) %>% 
  summarize(pe_mean = mean(p_encounter)*100,
            pe_L90 = quantile(p_encounter, 0.05)*100,
            pe_U90 = quantile(p_encounter, 0.95)*100) %>% 
  mutate(pe_diff = pe_mean - lag(pe_mean)) 

mrsumm 

(change_per_level <- paste0(round((mean(mrsumm$pe_diff, na.rm=TRUE)), 2),'%'))

plm <- lm(pe_mean ~ vari, data=mrsumm)
(plmsumm <- summary(plm))
(lm_pval <- plmsumm$coefficients[2,4] %>% round(3))

minmax_pval <- pval(val1=mrs$p_encounter[mrs$vari == min(mrs$vari)],
                    val2=mrs$p_encounter[mrs$vari == max(mrs$vari)])$p1

mrsumm <- 
  mrsumm %>% 
  mutate(pe_mean = paste0(round(pe_mean, 2),'%'),
            pe_L90 = paste0(round(pe_L90,2),'%'),
            pe_U90 = paste0(round(pe_U90,2),'%')) %>% 
  select(vari:pe_U90)

print(mrsumm %>% select(vari:pe_U90))
message('Percent change per level = ',change_per_level)
message('Min Max p value = ', minmax_pval) 
message('Regression p value = ', lm_pval)
}
